 个人艺术家官网制作需求文档 1.1 项目定位 制作一个面向收藏家、策展人、画廊、媒体和普通观众的个人艺术家官方网站。 网站整体风格参考 Jeff Koons 官网 https://www.jeffkoons.com/ 但不复制其图片、文字、Logo、版式细节，仅参考其“艺术档案型官网”的信息结构和极简展示方式。 网站应体现： 高级、简洁、艺术档案感、国际化、作品优先。 重点是让访问者快速看到艺术家的作品、展览经历、个人履历、媒体资料和联系方式。 1.2 视觉风格要求 2.1 整体风格 网站以白色或浅灰背景为主，黑色文字，少量灰色辅助线。整体应有大量留白，避免复杂装饰。 页面气质参考博物馆、画廊、艺术家档案网站，而不是商业电商网站。 2.2 色彩 主色： 背景：#FFFFFF 主文字：#111111 次级文字：#666666 分割线：#E5E5E5 悬停状态：#999999 或轻微透明变化 禁止使用过多渐变、霓虹色、大面积彩色按钮。 2.3 字体 英文推荐： Helvetica Neue Arial Neue Haas Grotesk Inter 中文推荐： 思源黑体 Noto Sans SC 苹方 Microsoft YaHei 字体风格应简洁、现代、干净。 2.4 图片风格 作品图片是网站视觉核心。 要求： 图片展示比例统一，但不能强行裁切重要作品内容。 作品图可使用瀑布流、网格、横向大图展示。 鼠标悬停作品时，可显示作品名称、年份。 作品详情页图片应尽量大，信息放在图片下方或右侧。 1.3 网站结构 参考 Jeff Koons 官网栏目，网站建议设置以下页面。参考站的主导航包含 Artwork、Exhibitions、Biography、Bibliography、Shop，首页和页脚也保留这些入口。 本项目建议导航为： Home / 首页 Works / 作品 Exhibitions / 展览 Biography / 履历 Press / 文献与媒体 Contact / 联系 4. 页面详细需求 4.1 首页 Home 首页目标：快速建立艺术家形象，让用户第一眼看到姓名、代表作品和艺术家简介。 A. 顶部导航 固定或非固定均可，建议顶部横向导航。 桌面端： 左侧：艺术家姓名 / Logo 右侧：Works、Exhibitions、Biography、Press、Shop、Contact 移动端： 右上角菜单 点击后展开全屏或侧边菜单 B. 首屏区域 内容： 艺术家姓名，大字号显示 一张代表作品大图，或 3–6 张代表作品组成的视觉网格 可选一句艺术家宣言，例如： “Contemporary Artist based in Beijing / Singapore / New York” 参考站首页以艺术家姓名和多张图片为核心，并在后方放置艺术家简介内容。 C. 代表作品模块 标题：Selected Works 展示 6–12 件代表作品。 每个作品卡片包含： 作品图片 作品名称 年份 材料/媒介 点击进入作品详情页 D. 简介模块 标题：About the Artist 放一段 150–300 字简介。 内容包括： 艺术家身份 创作方向 主要媒介 作品主题 所在城市 展览/收藏经历概述 E. 最新动态模块 标题：News / Current 内容： 最近展览 新作品发布 媒体报道 合作项目 限量作品发售 F. 页脚 包含： Home Works Biography Exhibitions Press Contact Instagram / 小红书 / 微信 / 邮箱 Copyright 参考站页脚也包含主要栏目与社交平台入口。 4.2 作品页 Works 作品页是网站重要的页面。 参考站 Artwork 页面有 Featured 作品区，并且按照不同作品系列进行分类展示，例如 Early Works、Inflatables、The New、Equilibrium 等。 本网站作品页需求： A. 页面标题 B. 筛选功能 顶部提供作品分类筛选： All Painting Sculpture Installation Digital Art Photography Series A Series B Series C 分类名称由艺术家后期后台自定义。 C. 作品网格 默认 3 列网格，移动端 1 列。 每个作品卡片显示： 图片 作品名称 年份 鼠标悬停： 图片轻微变暗 显示作品名称、年份、媒介 点击进入详情页。 D. 作品详情页 详情页字段： 作品名称 年份 材料 / 媒介 尺寸 版本信息，如 Edition of 5，可选 作品状态，如 Available / Sold / Private Collection，可选 作品说明 高清作品图片，可多张 展览记录，可选 文献记录，可选 相关作品推荐 页面布局： 桌面端： 左侧大图，右侧作品信息 或上方大图，下方文字信息 移动端： 图片在上，文字在下 功能： 图片点击可放大查看 支持左右切换多张图片 返回 Works 按钮 上一件 / 下一件作品按钮，可选 4.3 展览页 Exhibitions 参考站展览页分为 Solo Exhibitions 与 Group Exhibitions，并按年代区间归档，如 2020–Present、2010–2019、2000–2009 等。 本网站展览页需求： A. 页面标题 Exhibitions B. 分类 分为： Solo Exhibitions / 个展 Group Exhibitions / 群展 Art Fairs / 艺术博览会，可选 C. 时间分组 按年份倒序排列。 示例： 2027 2026 2023-2025 D. 展览条目字段 每条展览包含： 展览名称 展览类型：个展 / 群展 / 艺博会 展览地点 城市 / 国家 开始日期 结束日期 主办机构 展览简介，可选 展览海报或现场图，可选 外部链接，可选 E. 展览详情页，可选 4.4 履历页 Biography 参考站 Biography 页面包含艺术家奖项、荣誉、履历信息，并按年份整理。 本网站履历页需求： A. 艺术家简介 包含： 中文简介 英文简介 艺术家照片 出生地，可选 现居城市 教育背景 创作媒介 创作主题 B. Artist Statement 艺术家自述，建议 500–1000 字。 内容可以包括： 为什么创作 作品关注什么议题 使用什么材料 个人艺术语言 未来创作方向 C. CV 履历 按照年份倒序排列： Education / 教育经历 Solo Exhibitions / 个展 Group Exhibitions / 群展 Awards / 奖项 Collections / 收藏 Residencies / 驻留 Public Projects / 公共项目 D. 下载功能（可选） 提供： 下载艺术家 CV PDF 下载作品集 PDF 下载 Press Kit，可选 4.5 文献与媒体页 Press / Bibliography 参考站 Bibliography 页面按出版物、展览图录、文章等资料进行归档。 本网站建议命名为 Press，更适合个人艺术家和媒体使用。 A. 页面标题 Press B. 内容分类 Articles / 媒体文章 Interviews / 采访 Publications / 出版物 Catalogues / 展览图录 Videos / 视频 Downloads / 下载资料 C. 条目字段 每条资料包含： 标题 作者 媒体 / 出版机构 日期 类型 简介 外部链接 封面图，可选 D. 展示方式 列表为主，不需要复杂卡片。 推荐格式： 2026 — Interview with [Artist Name], Art Magazine, Beijing 点击后可打开外部链接或详情页。 4.6 联系页 Contact 页面内容： 艺术家工作室邮箱 商务合作邮箱 画廊代理信息，可选 Instagram 小红书 微信二维码，可选 联系表单 联系表单字段： Name Email Subject Message Inquiry Type：收藏咨询 / 展览合作 / 媒体采访 / 其他 提交后： 发送邮件到指定邮箱 后台保存表单记录 用户看到提交成功提示 5. 后台管理需求 CMS 需要做一个后台，方便艺术家自己上传内容。 5.1 作品管理 可新增、编辑、删除作品。 字段： 作品名称 英文名称 年份 分类 系列 材料 尺寸 版本 价格，可选 状态：展示中 / 隐藏 / 已售 / 可询价 作品说明 多张图片上传 排序权重 SEO 标题 SEO 描述 5.2 展览管理 字段： 展览名称 类型：个展 / 群展 / 艺博会 年份 开始时间 结束时间 地点 城市 国家 简介 图片 外部链接 是否置顶 5.3 媒体资料管理 字段： 标题 类型 媒体名称 作者 日期 链接 简介 图片 5.4 简介管理 可编辑： 中文简介 英文简介 Artist Statement CV PDF 作品集 PDF 艺术家头像 / 肖像照 6. 响应式适配 必须适配： 桌面端：1440px、1920px 笔记本：1280px 平板：768px 手机：375px、390px、430px 移动端要求： 导航变为菜单 作品网格变为单列 图片优先显示 字体不能过小 联系按钮清晰可点 7. 交互要求 7.1 基础交互 导航悬停有轻微颜色变化 作品图片悬停显示信息 图片可点击放大 页面切换平滑，但不要过度动画 返回顶部按钮，可选 7.2 图片查看器 作品详情页支持： 点击图片放大 左右切换 ESC 关闭 移动端手势滑动，可选 7.3 加载体验 图片较多时要做懒加载。 首屏图片优先加载，其余图片滚动到附近再加载。 8. 多语言需求 建议支持中英文双语。 URL 结构： 中文： /zh/works 英文： /en/works 或默认英文，中文（其他语言看需求）作为切换版本。 每个作品、展览、简介都需要支持中英文字段。 9. SEO 要求 每个页面需要支持： Title Description Open Graph 图片 页面关键词 作品详情页独立 SEO 自动生成 sitemap.xml robots.txt 404 页面 示例： [Artist Name] | Contemporary Artist 作品详情页： [Artwork Name], [Year] | [Artist Name] 10. 性能要求 首页首屏加载时间控制在 3 秒以内 图片自动压缩为 WebP 支持高清原图，但前台展示压缩图 使用懒加载 避免一次性加载所有作品高清图 手机端图片单张建议小于 500KB 桌面端大图建议小于 1.5MB 11. 技术建议 方案 A：轻量展示站 建议技术： Next.js / Nuxt / Astro 后台 CMS：Sanity / Strapi / Payload CMS / WordPress Headless 图片存储：Cloudinary / 本地服务器 / 对象存储 部署：Vercel / Netlify / 云服务器 方案 B：WordPress 方案 要求： 自定义主题 不使用廉价模板感很强的页面 后台可管理作品、展览、文章 支持中英文 支持图片压缩 方案 C：完全定制 适合长期品牌官网。 建议： 前端：Next.js 后端：Node.js / Strapi / Payload CMS 数据库：PostgreSQL 图片：对象存储 + CDN 部署：Vercel + 云数据库
